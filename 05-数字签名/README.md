# 数字签名

## 概念

数字签名是密码学中实现**身份认证**和**不可否认性**的核心技术。

```
签名过程：
消息 ──▶ [哈希] ──▶ 摘要 ──▶ [私钥签名] ──▶ 签名值

验证过程：
消息 ──▶ [哈希] ──▶ 摘要 ─┐
                          ├─▶ [公钥验证] ──▶ 通过/拒绝
签名值 ─────────────────┘
```

## 签名算法对比

| 算法 | 基于 | 签名长度 | 密钥长度 | 速度 | 状态 |
|------|------|---------|---------|------|------|
| **RSA-PSS** | 大数分解 | 256-512 B | 2048+ bit | 慢 | ✅ 广泛使用 |
| **ECDSA** | 椭圆曲线 | 64 B | 256 bit | 快 | ✅ 推荐 |
| **Ed25519** | 椭圆曲线 | 64 B | 256 bit | 极快 | ✅ 推荐 |
| **SM2 签名** | 椭圆曲线 | 64 B | 256 bit | 快 | ✅ 国密 |
| **DSA** | 离散对数 | 可变 | 可变 | 中 | ⚠️ 淘汰 |

## ECDSA 算法

ECDSA（Elliptic Curve Digital Signature Algorithm）是最常用的签名算法。

### 签名生成

```
1. 选择随机数 k
2. 计算 (x1, y1) = k × G
3. r = x1 mod n
4. s = k⁻¹(z + r·d) mod n
   其中 z = 消息哈希，d = 私钥
5. 签名 = (r, s)
```

### 签名验证

```
1. 计算 w = s⁻¹ mod n
2. 计算 u1 = z·w mod n
3. 计算 u2 = r·w mod n
4. 计算 (x1, y1) = u1×G + u2×Q  (Q = 公钥)
5. 验证 r ≡ x1 mod n
```

## Ed25519 签名

Ed25519 基于 Curve25519，由 Daniel Bernstein 设计：

- **优势**：签名速度快、实现简单、抗侧信道
- **签名长度**：64 字节
- **密钥长度**：32 字节私钥 + 32 字节公钥
- **广泛使用**：SSH、TLS、Signal 协议

## SM2 国密签名

SM2 签名是中国国密标准，基于椭圆曲线：

- 曲线参数与 NIST 不同（自定义曲线）
- 签名格式：(r, s) 各 256 bit
- 支持加密和签名两种功能
- 用于电子政务、金融等领域

## 数字证书（X.509）

数字证书将公钥与身份绑定：

```
X.509 证书结构：
┌─────────────────────────┐
│ 版本号                    │
│ 序列号                    │
│ 签名算法                  │
│ 颁发者 (CA)               │
│ 有效期                    │
│ 使用者 (Subject)           │
│ 使用者公钥                 │
│ 扩展信息                   │
│ CA 的数字签名              │
└─────────────────────────┘
```

### 证书链

```
根 CA (自签名)
  │
  ▼
中间 CA (根CA签名)
  │
  ▼
终端实体证书 (中间CA签名)
  │
  ▼
服务器/客户端
```

## 不可否认性

数字签名提供不可否认性：

1. Alice 用私钥签名消息
2. Bob 用 Alice 的公钥验证签名
3. 只有 Alice 持有私钥，所以 Alice 不能否认签名
4. 签名时间戳可证明签名时间
