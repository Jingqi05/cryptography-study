# 后量子密码学（PQC）

## 什么是后量子密码学

后量子密码学（Post-Quantum Cryptography）是能够抵抗量子计算机攻击的密码算法。

### 量子计算的威胁

```
经典计算机                    量子计算机
    │                            │
    ▼                            ▼
  RSA-2048                     Shor 算法
  破解需 10^20 年              可在数小时内破解
    │                            │
    ▼                            ▼
  ECC-256                      Shor 算法
  破解需 10^15 年              可在数小时内破解
    │                            │
    ▼                            ▼
  AES-256                      Grover 算法
  安全 256 bit                 降为 128 bit → 仍安全
```

**关键结论**：
- 非对称加密（RSA/ECC/DH）→ 量子计算机可破解
- 对称加密（AES）→ 密钥加倍即可抵抗
- 哈希算法 → 输出加倍即可抵抗

## NIST PQC 标准（2024）

2024 年 8 月，NIST 发布首批后量子密码标准：

| 标准 | 算法 | 原名 | 用途 | 数学基础 |
|------|------|------|------|---------|
| **FIPS 203** | ML-KEM | CRYSTALS-Kyber | 密钥封装 | 格密码 |
| **FIPS 204** | ML-DSA | CRYSTALS-Dilithium | 数字签名 | 格密码 |
| **FIPS 205** | SLH-DSA | SPHINCS+ | 数字签名 | 哈希 |

### ML-KEM（密钥封装）

```
密钥封装机制 (KEM)：
  1. 接收方生成公私钥对 (pk, sk)
  2. 发送方用 pk 封装 → (密文 c, 共享密钥 K)
  3. 接收方用 sk 解封 c → 共享密钥 K

等价于：密钥交换协议
```

| 参数集 | 公钥长度 | 密文长度 | 共享密钥长度 |
|--------|---------|---------|------------|
| ML-KEM-512 | 800 B | 768 B | 32 B |
| ML-KEM-768 | 1184 B | 1088 B | 32 B |
| ML-KEM-1024 | 1568 B | 1568 B | 32 B |

### ML-DSA（数字签名）

| 参数集 | 公钥长度 | 签名长度 | 安全级别 |
|--------|---------|---------|---------|
| ML-DSA-44 | 1312 B | 2420 B | NIST 2 |
| ML-DSA-65 | 1952 B | 3293 B | NIST 3 |
| ML-DSA-87 | 2592 B | 4595 B | NIST 5 |

### SLH-DSA（基于哈希的签名）

- 不依赖格密码，安全性基于哈希函数
- 签名较大但信任假设最小
- 作为后备方案

## 后量子密码的数学基础

### 格密码（Lattice-based）

```
最近向量问题 (CVP)：
  给定格 L 和目标点 t，找格中离 t 最近的点

最短向量问题 (SVP)：
  给定格 L，找格中最短非零向量

这些问题在经典和量子计算机上都被认为是困难的
```

### 基于编码的密码

- McEliece 加密方案（1978）
- 基于纠错码的解码困难问题

### 基于多变量的密码

- 多变量二次方程组求解困难
- 签名方案

## 迁移策略

### Mosca 定理

```
如果 t₁ + t₂ > t₃，则需要立即开始迁移

t₁ = 系统迁移所需时间
t₂ = 数据需要保密的时间
t₃ = 量子计算机可用的时间
```

### 混合模式

过渡期推荐使用经典 + 后量子混合：

```
TLS 1.3 + ML-KEM：
  ECDHE (经典) + ML-KEM (后量子)
  两者共享密钥取并集
  任一安全即可保证整体安全
```
