# 对称加密

## 概念

对称加密使用**同一把密钥**进行加密和解密。发送方和接收方必须共享同一个密钥。

```
明文 ──▶ [加密] ──▶ 密文 ──▶ [解密] ──▶ 明文
         密钥K                      密钥K
```

**优点**：速度快，适合大量数据加密
**缺点**：密钥分发困难，n 个通信方需要 n(n-1)/2 个密钥

## 算法对比

| 算法 | 密钥长度 | 分组长度 | 类型 | 状态 | 说明 |
|------|---------|---------|------|------|------|
| **AES** | 128/192/256 bit | 128 bit | 分组密码 | ✅ 推荐 | 全球最广泛使用 |
| **DES** | 56 bit | 64 bit | 分组密码 | ❌ 已淘汰 | 密钥太短，可暴力破解 |
| **3DES** | 168 bit | 64 bit | 分组密码 | ⚠️ 淘汰中 | DES 三次加密，性能差 |
| **SM4** | 128 bit | 128 bit | 分组密码 | ✅ 国密 | 中国国密标准 |
| **ChaCha20** | 256 bit | — | 流密码 | ✅ 推荐 | Google 推广，移动设备性能优异 |
| **RC4** | 可变 | — | 流密码 | ❌ 已淘汰 | TLS 中已禁用 |

## AES 算法详解

AES（Advanced Encryption Standard）是最常用的对称加密算法。

### AES 工作流程

```
明文 (128 bit)
    │
    ▼
┌─────────────┐
│  轮密钥加    │  ← 初始轮密钥
├─────────────┤
│  字节替换     │  ← S-Box 查表
│  行移位       │  ← 每行左移不同位数
│  列混合       │  ← GF(2^8) 上的矩阵乘法
│  轮密钥加    │  ← 轮密钥
├─────────────┤
│  ... (重复)   │  ← 10/12/14 轮
├─────────────┤
│  字节替换     │
│  行移位       │  ← 最后一轮无列混合
│  轮密钥加    │
└─────────────┘
    │
    ▼
密文 (128 bit)
```

| AES 版本 | 密钥长度 | 轮数 |
|---------|---------|------|
| AES-128 | 128 bit | 10 轮 |
| AES-192 | 192 bit | 12 轮 |
| AES-256 | 256 bit | 14 轮 |

## 分组密码工作模式

| 模式 | 全称 | 特点 | 推荐 |
|------|------|------|------|
| **ECB** | 电子密码本 | 每块独立加密，相同明文→相同密文 | ❌ 不推荐 |
| **CBC** | 密码分组链接 | 需要 IV，块间有依赖 | ⚠️ 一般 |
| **CTR** | 计数器模式 | 可并行，变成流密码 | ✅ 推荐 |
| **GCM** | Galois/计数器 | 认证加密（AEAD），同时加密+校验 | ✅ 强烈推荐 |
| **CCM** | CBC-MAC 计数器 | AEAD，用于 WiFi WPA2 | ✅ 推荐 |

### ECB vs CBC vs CTR 对比

```
ECB 模式（不安全）：
P1 ──▶ E(K) ──▶ C1
P2 ──▶ E(K) ──▶ C2    ← 相同明文产生相同密文
P3 ──▶ E(K) ──▶ C3

CBC 模式：
P1 ──▶ ⊕ ──▶ E(K) ──▶ C1
       ↑IV
P2 ──▶ ⊕ ──▶ E(K) ──▶ C2
       ↑C1
P3 ──▶ ⊕ ──▶ E(K) ──▶ C3
       ↑C2

CTR 模式：
Counter1 ──▶ E(K) ──▶ ⊕ ──▶ C1
Counter2 ──▶ E(K) ──▶ ⊕ ──▶ C2    ← 可并行
Counter3 ──▶ E(K) ──▶ ⊕ ──▶ C3
```

## GCM 认证加密（AEAD）

GCM 是目前最推荐的工作模式，同时提供**加密**和**认证**：

```
明文 + AAD ──▶ [AES-GCM] ──▶ 密文 + 认证标签 (Tag)
                  密钥 + IV
```

- **加密**：将明文转为密文
- **认证**：Tag 验证数据未被篡改
- **AAD**：附加认证数据（不加密但受保护，如 TLS 头部）

## SM4 国密算法

SM4 是中国国密标准的分组密码算法，用于无线局域网等领域。

| 参数 | 值 |
|------|-----|
| 密钥长度 | 128 bit |
| 分组长度 | 128 bit |
| 轮数 | 32 轮 |
| 结构 | 非平衡 Feistel 网络 |

## ChaCha20 流密码

ChaCha20 由 Daniel Bernstein 设计，Google 在 TLS 中推广：

- 256 bit 密钥，64 bit nonce
- 软件实现比 AES 更快（无 AES-NI 硬件时）
- 抗侧信道攻击
- 与 Poly1305 MAC 配合使用（AEAD）
