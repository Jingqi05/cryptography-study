# 密钥管理

## 密钥生命周期

```
生成 → 分发 → 使用 → 存储 → 轮换 → 销毁
```

| 阶段 | 说明 |
|------|------|
| 生成 | 使用 CSPRNG 生成随机密钥 |
| 分发 | 通过安全通道传输密钥 |
| 使用 | 加解密、签名验证 |
| 存储 | HSM、KMS、安全 enclave |
| 轮换 | 定期更换密钥 |
| 销毁 | 安全擦除密钥材料 |

## 密钥派生函数（KDF）

### HKDF（HMAC-based KDF）

```
HKDF-Extract(salt, IKM) → PRK
HKDF-Expand(PRK, info, L) → OKM
```

用于从共享密钥派生会话密钥，TLS 1.3 使用 HKDF。

### PBKDF2

```
DK = PBKDF2(PRF, Password, Salt, c, dkLen)
  c = 迭代次数（建议 100,000+）
```

用于从用户密码派生密钥。

### Argon2（推荐）

2015 年密码哈希竞赛冠军：

- 抗 GPU/ASIC 攻击
- 可调参数：内存、迭代次数、并行度
- 推荐用于密码存储

## 密钥交换协议

| 协议 | 说明 | 前向保密 |
|------|------|---------|
| RSA 密钥传输 | 用接收方公钥加密密钥 | ❌ |
| DH 密钥交换 | 经典 Diffie-Hellman | ❌ 静态 |
| ECDHE | 临时椭圆曲线 DH | ✅ |
| X25519 | Curve25519 DH | ✅ |

## 前向保密（PFS）

前向保密确保即使长期私钥泄露，过去的会话密钥仍然安全：

```
传统 RSA 密钥传输：
  私钥泄露 → 所有历史会话可解密 ❌

ECDHE 前向保密：
  每次会话生成临时密钥对
  会话结束销毁临时私钥
  长期私钥泄露 → 历史会话仍安全 ✅
```

TLS 1.3 强制使用 ECDHE，不再支持 RSA 密钥传输。
