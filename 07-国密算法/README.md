# 国密算法

## 概述

国密算法是中国国家密码管理局发布的密码学标准，用于替代国际算法，保障国家信息安全。

| 算法 | 类型 | 用途 | 对应国际算法 |
|------|------|------|------------|
| **SM2** | 非对称加密/签名 | 密钥交换、数字签名、加密 | RSA/ECC |
| **SM3** | 哈希算法 | 完整性校验、数字签名 | SHA-256 |
| **SM4** | 对称加密 | 数据加密 | AES |
| **SM9** | 标识密码 | 基于身份的加密/签名 | IBE |

### SM2 vs ECDSA

| 对比 | SM2 | ECDSA |
|------|-----|-------|
| 曲线 | 自定义曲线 | NIST P-256 |
| 签名速度 | 略快 | 中等 |
| 安全性 | 256 bit | 256 bit |
| 应用 | 中国政务/金融 | 全球通用 |

## SM3 哈希算法

### 算法结构

```
消息 M
  │
  ▼
填充 (使长度 ≡ 448 mod 512)
  │
  ▼
512 bit 分组
  │
  ▼
64 轮压缩函数
  │
  ▼
256 bit 哈希值
```

### 与 SHA-256 对比

| 对比 | SM3 | SHA-256 |
|------|-----|---------|
| 输出长度 | 256 bit | 256 bit |
| 分组长度 | 512 bit | 512 bit |
| 轮数 | 64 | 64 |
| 布尔函数 | FF/GG | Ch/Maj |
| 置换函数 | P0/P1 | σ0/σ1 |
| 安全性 | 相当 | 相当 |

## SM4 分组密码

### 算法参数

| 参数 | 值 |
|------|-----|
| 密钥长度 | 128 bit |
| 分组长度 | 128 bit |
| 轮数 | 32 轮 |
| 结构 | 非平衡 Feistel 网络 |

### 加密流程

```
明文 (128 bit) = (X0, X1, X2, X3)
  │
  ▼
32 轮 Feistel 迭代
  │  每轮：Xi+4 = Xi ⊕ T(Xi+1 ⊕ Xi+2 ⊕ Xi+3 ⊕ rk_i)
  │  T = 非线性变换 τ ∘ L
  ▼
密文 (128 bit) = (Y0, Y1, Y2, Y3) = (X35, X34, X33, X32)
```

### 工作模式

SM4 支持 ECB/CBC/CTR/CFB/OFB/GCM 等标准模式。

## SM9 标识密码

SM9 是基于身份的密码学（IBC），用户的公钥就是其身份标识（如邮箱、手机号）。

```
传统 PKI：
  用户 → 证书 → CA → 公钥
  需要证书管理

SM9 IBC：
  用户 → 身份标识（邮箱）= 公钥
  由密钥生成中心 (KGC) 生成私钥
```

### SM9 应用

| 功能 | 说明 |
|------|------|
| 密钥交换 | 基于身份的密钥协商 |
| 数字签名 | SM9 签名算法 |
| 加密 | SM9 加密算法 |
| 密钥封装 | SM9 密钥封装 |

## 国密应用场景

| 场景 | 使用算法 |
|------|---------|
| SSL VPN | SM2 + SM3 + SM4 |
| 电子签章 | SM2 签名 |
| 金融交易 | SM4-GCM 加密 |
| 身份认证 | SM9 标识密码 |
| 区块链 | SM2 + SM3 |
| 信创替代 | 全栈国密 |
