# 哈希算法

## 概念

哈希函数将任意长度的输入映射为固定长度的输出（摘要/指纹）。

```
H("Hello")           → 185f8db32271fe25f561a6fc938b2e26...
H("Hello World!")    → 7f83b1657ff1fc53b92dc18148a1d65d...
H(任意长文件)         → 256 bit 固定输出
```

## 核心特性

| 特性 | 说明 |
|------|------|
| **确定性** | 相同输入 always 产生相同输出 |
| **抗原像攻击** | 给定 H(x)，无法反推 x |
| **抗第二原像攻击** | 给定 x，找不到另一个 x' 使 H(x) = H(x') |
| **抗碰撞攻击** | 极难找到 x ≠ x' 使 H(x) = H(x') |
| **雪崩效应** | 输入改变 1 bit，输出变化约 50% |

## 算法对比

| 算法 | 输出长度 | 安全性 | 速度 | 用途 |
|------|---------|--------|------|------|
| **MD5** | 128 bit | ❌ 已破解 | 快 | 仅文件校验 |
| **SHA-1** | 160 bit | ❌ 已破解 | 中 | 遗留系统 |
| **SHA-256** | 256 bit | ✅ 安全 | 中 | Bitcoin, TLS |
| **SHA-512** | 512 bit | ✅ 安全 | 中 | 高安全场景 |
| **SHA-3** | 可变 | ✅ 安全 | 中 | 新一代标准 |
| **SM3** | 256 bit | ✅ 安全 | 中 | 中国国密 |
| **BLAKE2** | 可变 | ✅ 安全 | 快 | 通用高性能 |
| **BLAKE3** | 可变 | ✅ 安全 | 极快 | 2024 新兴 |

## SHA-256 算法

### 工作流程

```
消息 M
  │
  ▼
填充 (使长度 ≡ 448 mod 512)
  │
  ▼
附加消息长度 (64 bit)
  │
  ▼
分成 512 bit 块
  │
  ▼
对每个块进行 64 轮压缩
  │
  ▼
输出 256 bit 哈希值
```

### 初始哈希值

SHA-256 使用 8 个 32 bit 初始值（前 8 个素数的平方根小数部分）：

```
H0 = 6a09e667  H1 = bb67ae85
H2 = 3c6ef372  H3 = a54ff53a
H4 = 510e527f  H5 = 9b05688c
H6 = 1f83d9ab  H7 = 5be0cd19
```

## SHA-3（Keccak）

SHA-3 与 SHA-2 完全不同的结构，基于**海绵结构**（Sponge Construction）：

```
输入消息
    │
    ▼
┌──────────────────────┐
│   吸收阶段 (Absorb)   │  ← 消息块依次异或到状态
│   1600 bit 状态       │
│   24 轮置换函数       │
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│   挤出阶段 (Squeeze)  │  ← 从状态中提取输出
│   256/512 bit 输出    │
└──────────────────────┘
```

### SHA-3 变体

| 变体 | 输出长度 | 安全级别 |
|------|---------|---------|
| SHA3-224 | 224 bit | 112 bit |
| SHA3-256 | 256 bit | 128 bit |
| SHA3-384 | 384 bit | 192 bit |
| SHA3-512 | 512 bit | 256 bit |
| SHAKE128 | 可变 | 128 bit |
| SHAKE256 | 可变 | 256 bit |

## SM3 国密哈希

SM3 是中国国密标准的哈希算法，输出 256 bit。

- 消息填充 → 512 bit 分组
- 64 轮压缩函数
- 使用布尔函数 FF/GG 和置换函数 P0/P1
- 安全性与 SHA-256 相当

## 哈希算法应用

### 1. 密码存储

```
用户密码 "123456"
    │
    ▼
加盐 (salt = random)
    │
    ▼
bcrypt/scrypt/Argon2 (慢哈希)
    │
    ▼
存储：hash + salt
```

**永远不要**用 MD5/SHA-256 直接存储密码，必须使用慢哈希函数。

### 2. 数字签名

```
消息 ──▶ [哈希] ──▶ 摘要 ──▶ [私钥签名] ──▶ 签名
```

### 3. 区块链（Merkle Tree）

```
         Root Hash
        /         \
    Hash(AB)    Hash(CD)
    /    \      /    \
  H(A)  H(B)  H(C)  H(D)
```

### 4. HMAC（消息认证码）

```
HMAC(K, M) = H((K ⊕ opad) || H((K ⊕ ipad) || M))
```

用于验证消息完整性和真实性。
