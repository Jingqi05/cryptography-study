# 非对称加密

## 概念

非对称加密使用**一对密钥**：公钥（public key）和私钥（private key）。公钥可公开，私钥必须保密。

```
加密：明文 ──▶ [公钥加密] ──▶ 密文 ──▶ [私钥解密] ──▶ 明文
签名：消息 ──▶ [私钥签名] ──▶ 签名 ──▶ [公钥验证] ──▶ 通过/拒绝
```

**优点**：解决密钥分发问题，支持数字签名
**缺点**：速度比对称加密慢 100-1000 倍

## 算法对比

| 算法 | 数学问题 | 密钥长度 | 安全等效 | 状态 |
|------|---------|---------|---------|------|
| **RSA** | 大整数分解 | 2048/4096 bit | — | ✅ 广泛使用 |
| **ECC** | 椭圆曲线离散对数 | 256 bit | ≈ RSA 3072 | ✅ 推荐 |
| **SM2** | 椭圆曲线 | 256 bit | ≈ RSA 2048 | ✅ 国密标准 |
| **Ed25519** | 椭圆曲线 | 256 bit | ≈ RSA 3072 | ✅ 签名推荐 |
| **DSA** | 离散对数 | 可变 | — | ⚠️ 淘汰中 |

## RSA 算法

### 密钥生成

```
1. 选择两个大素数 p, q
2. 计算 n = p × q
3. 计算 φ(n) = (p-1)(q-1)
4. 选择 e，满足 1 < e < φ(n) 且 gcd(e, φ(n)) = 1
5. 计算 d = e⁻¹ mod φ(n)

公钥：(n, e)
私钥：(n, d)
```

### 加解密

```
加密：C = M^e mod n
解密：M = C^d mod n
```

### 安全性基础

RSA 的安全性基于**大整数分解问题**：给定 n = p × q，求 p 和 q 在计算上是困难的。

| 密钥长度 | 安全级别 | 状态 |
|---------|---------|------|
| 1024 bit | 80 bit | ❌ 已不安全 |
| 2048 bit | 112 bit | ✅ 当前标准 |
| 3072 bit | 128 bit | ✅ 推荐 |
| 4096 bit | 140 bit | ✅ 高安全 |

## 椭圆曲线密码学（ECC）

### 椭圆曲线方程

```
y² = x³ + ax + b  (mod p)
```

### 椭圆曲线上的运算

```
点加法：P + Q = R
  1. 连接 P 和 Q 的直线
  2. 与曲线交于第三点
  3. 关于 x 轴对称得到 R

标量乘法：kP = P + P + ... + P (k 次)
  - 已知 k 和 P，计算 Q = kP（容易）
  - 已知 Q 和 P，求 k（困难）→ 椭圆曲线离散对数问题
```

### ECC 优势

| 对比 | RSA-3072 | ECC-256 |
|------|----------|---------|
| 密钥长度 | 3072 bit | 256 bit |
| 安全级别 | 128 bit | 128 bit |
| 签名速度 | 较慢 | 较快 |
| 密钥生成 | 慢 | 快 |
| 带宽 | 大 | 小 |

## Diffie-Hellman 密钥交换

### 经典 DH

```
公开参数：g, p

Alice                           Bob
  │                               │
  │ 选择私钥 a                    │ 选择私钥 b
  │ 计算 A = g^a mod p            │ 计算 B = g^b mod p
  │                               │
  │──── A ──────────────────────▶│
  │◀───────────────────── B ─────│
  │                               │
  │ 计算 s = B^a mod p            │ 计算 s = A^b mod p
  │                               │
  │     s = g^(ab) mod p = s      │  ← 共享密钥
```

### ECDH（椭圆曲线 DH）

```
公开参数：曲线 E, 基点 G

Alice                           Bob
  │ 选择私钥 a                    │ 选择私钥 b
  │ 计算 A = aG                   │ 计算 B = bG
  │                               │
  │──── A ──────────────────────▶│
  │◀───────────────────── B ─────│
  │                               │
  │ 计算 S = aB = abG             │ 计算 S = bA = abG
```

ECDH 是 TLS 1.3 中的密钥交换标准。

## 核心应用

| 应用 | 算法 | 说明 |
|------|------|------|
| 密钥交换 | ECDHE | TLS 1.3 前向保密 |
| 数字签名 | ECDSA, EdDSA | 身份认证、不可否认 |
| 密钥封装 | RSA-OAEP, ECIES | 安全传输对称密钥 |
| 证书 | X.509 + RSA/ECC | PKI 信任体系 |
